# 안녕하세요. 웹퍼블리셔/프론트엔드 개발자 이자용 입니다.
시각 디자인과 출신으로 디자인에 대한 관심과 함께 효율적인 코드 작성에 중점을 두고 커리어를 이어가고 있고, 개인 깃헙과 블로그에 꾸준히 업로드하면서 내가 무엇을 공부했는지, 궁금해 한 것은 무엇이었는지 복기하는 방법으로 코드를 개선시켜 나가는 중입니다. 마크업 개발로 경력을 이어온 지는 어느덧 햇수로 10년차가 되었습니다. 그동안 다양한 프로젝트를 통해 마크업 개발자로 문제를 해결해왔습니다.

웹 접근성을 고려한 마크업 작성과 함께 효율적인 스타일 시트를 위한 고민을 하여 프론트단에서 성능을 최적화 하기 위해 노력하면서 gulp 같은 자동화 도구를 회사에 적용시키기도 하고, 꼭 마크업 개발에서 생긴 이슈 뿐만 아니라 팀원들과 함께 풀어낼 수 있는 이슈는 여러 이해관계를 가진 작업자들과의 소통으로 더 나은 방향으로 문제를 해결 할 수 있도록 업무를 처리해왔습니다. 마크업 개발자에서 프론트엔드 개발자로 전향하기 위해 꾸준히 공부하다가 최근에는 프론트엔드 부트캠프 항해99 파트타임 수료를 앞두고 있습니다.

프론트엔드 부트캠프에서는 18주간 파트타임으로 진행하여 자바스크립트와 리액트를 새롭게 배우고, 미니프로젝트(미니 쇼핑몰 : https://github.com/zadragon/hanghaeShop), 클론코딩프로젝트(유튜브 : https://github.com/zadragon/cloneTube), 실전 프로젝트(습관관리 서비스[진행중] : https://github.com/zadragon/godSaengLife) 등 node.js 백엔드개발자와 함께 프로젝트를 진행했습니다. 부트캠프를 통해 기술적인 면도 많이 배웠지만, 공부하는 습관을 익혔다는 점이 가장 큰 수확이라고 생각합니다.

UI개발을 위한 자바스크립트는 꾸준히 해왔던 작업이고, 이전 회사에서 NEXT.js 환경에서 쇼핑몰 검색기능에 소소한 기능을 작업을 시작으로 cra환경에서 마크업/ui개발 등을 경험하다가 본격적인 RESTful api 방식의 데이터 핸들링은 부트캠프를 통해 숙련도를 쌓아 나갔습니다. 다년간의 마크업 경력을 바탕으로 react상에서 구조적인 마크업과 폴더구조를 고려하여 마크업 개발이 가능합니다. 또 sass/styled-components와 tailwind를 통해 를 통해 자유자재로 스타일링이 가능합니다. 스크롤 이벤트 위주의 애플스타일의 ui를 구성하거나 트렐로같이 마우스 이벤트를 제작해보기도 했습니다.

개인프로젝트로 node.js와 python, mongo db를 이용한 간단한 웹사이트를 만들어본 경험이 있고, useState와 context API, redux / redux-toolkit 로 상태관리를 하고 firebase를 통해 직접 간단한 db연동해보기도 하고, aws를 통해 react빌드된 프로젝트를 s3환경에서 배포하고 도메인 연결이 가능합니다.
